## Cuándo leer cada documento

### Entro a un proyecto TouchDesigner
→ Lee `touchdesigner/td_onboarding.md` (cómo conectar, verificar, inventariar)
→ Lee `touchdesigner/td_operadores_lops.md` (catálogo de operadores LOPs)

### Me piden hacer algo y no sé cómo
→ Lee `touchdesigner/td_snippets.md` (código reutilizable)
→ Si el error es conocido: `touchdesigner/td_pitfalls.md`

### Me piden un shader GLSL
→ Lee `touchdesigner/shaders/glsl_knowledge.md` (conocimiento acumulado)
→ Lee `touchdesigner/shaders/glsl_patterns.md` (patrones reutilizables)

### Me planteo usar Claude Code LOP
→ Lee `touchdesigner/td_claude_code.md` antes de decidir

### Descubro algo nuevo trabajando
→ Identifica la categoría (pitfall, snippet, GLSL, workflow)
→ Lee el documento correspondiente del repo via GitHub MCP
→ Añade la sección nueva y haz commit

---

## Reglas

1. Leer el documento ANTES de editarlo (nunca desde memoria)
2. Documentos pequeños: si uno supera 15KB, dividirlo
3. Commit con formato: `[carpeta] acción breve`
4. No incluir nombres de usuario, rutas locales ni datos personales
